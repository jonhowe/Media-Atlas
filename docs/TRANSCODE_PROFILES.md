# Transcode Profiles

Media Atlas generates only app-defined `ffmpeg` commands. Users can choose from seeded profiles when creating a transcode plan. Outputs are staged under the configured transcode staging directory; source media is not replaced or deleted.

## Available Profiles

| Profile | Command family | Relative speed | Use case |
| --- | --- | --- | --- |
| Remux to MKV | stream copy | Very fast | Container changes without re-encoding. |
| HEVC Archive Balanced | `libx265 -crf 20 -preset medium` | Slow | Maximum compression and quality. |
| HEVC Archive Fast | `libx265 -crf 21 -preset fast` | Medium | Recommended default for most archival conversions. |
| HEVC Archive Faster | `libx265 -crf 22 -preset faster` | Fast | Bulk conversions where larger outputs are acceptable. |
| HEVC Quick Sync | `hevc_qsv -global_quality 24` | Very fast | Intel iGPU/Quick Sync hosts. |
| HEVC NVENC | `hevc_nvenc -cq 24 -preset p5` | Very fast | NVIDIA GPU hosts. |
| HEVC VAAPI | `hevc_vaapi -qp 24` | Very fast | Linux VAAPI hosts, usually Intel or AMD. |
| H.264 Compatibility | `libx264 -crf 20 -preset slow` | Medium | MP4 compatibility outputs. |
| Manual Review Only | none | n/a | Track complex files that should not produce a runnable command. |

## Software HEVC Profiles

`HEVC Archive Balanced` keeps the original archival behavior and favors compression efficiency over throughput.

`HEVC Archive Fast` is the recommended first-choice software encode profile. It uses a slightly higher CRF and faster preset than Balanced, so it should reduce encode time while keeping output quality practical for library archival use.

`HEVC Archive Faster` is useful for bulk conversions where throughput matters more than squeezing every possible byte out of the file.

## Hardware Profiles

Hardware profiles are optional. They remain visible because the same image can run on different hosts, but they require host and container device support.

For VAAPI or Intel Quick Sync on Linux, add device access to `docker-compose.yml`:

```yaml
services:
  media-atlas:
    devices:
      - /dev/dri:/dev/dri
```

Media Atlas checks for `/dev/dri/renderD128` before running VAAPI or Quick Sync jobs. If the device is not present inside the container, the item fails preflight instead of starting a doomed encode.

For NVIDIA NVENC, install the NVIDIA Container Toolkit on the host and add:

```yaml
services:
  media-atlas:
    gpus: all
```

## CPU And Encoder Checks

Useful Docker checks:

```bash
docker stats media-atlas
docker exec -it media-atlas nproc
docker inspect media-atlas
```

If `libx265` logs messages such as `No thread pool allocated`, check container CPU limits first. If the container sees the expected CPUs and x265 still underutilizes the host, compare the Fast and Faster profiles before adding encoder-specific threading parameters.

## Safety Notes

- Commands are generated by Media Atlas from profile templates.
- Source files are input-only.
- Existing staged target paths are not overwritten.
- Completed outputs are verified with `ffprobe`.
- Replacement of originals is intentionally outside the current production workflow.

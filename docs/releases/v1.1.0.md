# Media Atlas v1.1.0

Media Atlas v1.1.0 introduces a full retention-review workflow, richer transcode planning and diagnostics, embedded release metadata, and broad responsiveness improvements. This stable release consolidates the work published in `v1.1.0-beta1` through `v1.1.0-beta7`.

## Media Retention Review

- Added redacted Seerr, Sonarr, and Radarr connection management, including multiple Arr instances, per-service path mappings, Seerr service IDs, and optional environment-based connection seeds.
- Added retention analysis that correlates paginated Seerr requests, Arr inventory, Media Atlas scan data, and Plex watch history without deleting production media during analysis.
- Expanded analysis into an auditable review ledger with per-title, per-scope, and per-file decisions, including requested-season handling, partially available TV seasons, failed or unrouted source warnings, and compatible backfill of older retention snapshots.
- Added Retention UI and dashboard metrics, filtering, CSV export, analysis and action history, source warnings, and scoped handoff of eligible files to the Transcode Planner.
- Preserved whole-copy deletion safeguards. Deletion requires typed confirmation, fresh source revalidation, and immutable remediation audits; season-level review never authorizes Sonarr deletion.
- Added an optional 03:00 retention schedule that remains disabled by default.

## Logs and Transcode Planning

- Added a centralized Logs page for filtered application events, live transcode output, and scan diagnostics.
- Persisted application logs as daily UTC-rotated JSONL files under the configured logs directory while continuing to stream logs to stdout. Retention cleanup excludes the active log file.
- Expanded the Transcode Planner with Easy Win, Remux Only, and Review workflows; search and pagination; page selection; category-specific profile suggestions; override warnings; links from Quality Candidates; and explicit review-only plan state.

## Reliability and Plex Integration

- Kept the Web UI responsive during scans, Plex syncs, retention analysis, and transcodes by isolating blocking backend work, running synchronous API handlers in worker threads, and preventing overlapping UI polling requests.
- Fixed Plex TV-library syncs to fetch episode media parts so scanned episode files can be path-matched and enriched.
- Added regression coverage for TV and movie library listing behavior and for scan and Plex background jobs not blocking the request event loop.

## Version Metadata and Documentation

- Added authenticated `/api/version` metadata for the release version, git SHA, build date, and image tag, and surfaced the current release tag in the sidebar.
- Updated GHCR builds and smoke tests to embed and verify the GitHub release tag as the application version.
- Added a deterministic documentation demo generator, a ten-screen Web UI tour, synthetic screenshot assets, and refreshed development guidance.
- Updated the plan, execute, and resume workflows to pin Release Automator v0.4.1.

## Upgrade Notes

- This release includes SQLite migrations `0008_media_retention_review` and `0009_retention_evaluation_ledger`. Back up the database before upgrading. To roll back to an older image, restore the pre-upgrade database backup.
- Review `.env.docker.example`, `.env.example`, and the deployment and operations guides before enabling retention integrations or scheduling.
- Retention review is designed to be review-led and guarded. Configure and test connections with disposable or mocked services before taking remediation actions against production libraries.
